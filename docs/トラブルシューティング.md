# トラブルシューティング

公開後・運用時によくある問題と解決方法をまとめました。

---

## 🔴 表示の問題

### Q1: 公開URLにアクセスしても何も表示されない

**チェック1: Render のステータス確認**

1. [Render ダッシュボード](https://dashboard.render.com) を開く
2. `plush-puppy-coat-clinic` を選択
3. ステータスを確認:
   - 🟢 **Live**: 正常
   - 🔴 **Deploy failed**: ビルド失敗 → Logsタブでエラー確認
   - 🟡 **Suspended**: スリープ中（無料プラン） → アクセスすれば復帰

**チェック2: ブラウザの開発者ツール**

- F12キー → Console タブ
- 赤いエラーメッセージがあるかチェック

**チェック3: 直接URL確認**

- `https://plush-puppy-coat-clinic.onrender.com/index.html` を直接開いてみる
- ここで表示されない → Render 側の問題
- ここでは表示される → makeshop の埋め込み設定問題

---

### Q2: makeshop からは見られないが、Render URL では見られる

→ **iframe 埋め込みコードのURL違い** の可能性大

**確認手順:**
1. makeshopフリーページの編集画面を開く
2. 貼り付けたコードの中の `iframe src="..."` を確認
3. その URL を直接ブラウザで開いてみる
4. 404 や Cannot GET と出たら URL が間違っている

修正：`embed-snippet.html` の **2箇所** の `YOUR-RENDER-URL` を実際のURLに書き換え、makeshopに再貼り付け。

---

### Q3: iframe が表示されるが、中身が小さい/高さが足りない

→ **postMessage の Origin チェック不一致**

`embed-snippet.html` の JavaScript内:
```javascript
if (e.origin !== 'https://YOUR-RENDER-URL.onrender.com') return;
```

ここの URL が iframe の `src` と**完全一致**している必要があります。

- ❌ `https://plush-puppy-coat-clinic.onrender.com/` （末尾スラッシュあり）
- ✅ `https://plush-puppy-coat-clinic.onrender.com` （末尾スラッシュなし）

origin は protocol + domain のみで、パスやスラッシュは含みません。

---

### Q4: 商品画像が表示されない（プレースホルダーのまま）

**原因と対処:**

1. **インターネット接続** を確認
2. **`https://makeshop-multi-images.akamaized.net/` がブロックされていないか確認**
   - 社内ファイアウォールで CDN がブロックされている可能性
3. **該当商品が販売終了でないか** plushpuppyjapan.com で確認
4. ブラウザの開発者ツール（F12）→ Network タブで該当画像のステータスを確認
   - `404 Not Found` → 商品ID/画像URLが間違っている

## 🟡 動作の問題

### Q5: 「この内容で診断する」ボタンを押しても反応しない

**チェック手順:**

1. ブラウザの開発者ツール（F12）→ Console を開く
2. ボタンを押す
3. 赤いエラーが出るか確認

**よくある原因:**
- `index.html` のJavaScript構文エラー（編集後にカンマ忘れなど）
- Chart.js が読み込めていない（インターネット接続問題）

**直前の状態に戻す:**

```bash
git log --oneline  # 履歴確認
git reset --hard HEAD~1  # 1つ前のコミットに戻す
git push --force
```

---

### Q6: レーダーチャートが表示されない

**原因:** Chart.js が読み込めていない

**確認:**
- 開発者ツール → Console で `typeof Chart` と入力
- `undefined` なら未読込
- `function` なら正常

**対処:**
1. ネットワーク接続を確認
2. CDN へのアクセスが許可されているか確認:
   - `cdn.jsdelivr.net`
   - `unpkg.com`
3. それでもダメな場合: `assets/vendor/chart.umd.min.js` のローカル版が読み込まれるはず

---

### Q7: スマホで横スクロールが発生する

**原因と対処:**

1. **makeshop の親 CSS の干渉**
   - iframe 方式に変更（推奨）
2. **viewport meta の重複**
   - makeshop が自動で viewport を設定しているか確認
3. **iframe の width が固定値になっていないか**
   - `width:100%` が指定されているか確認

## 🟠 GitHub・Render の問題

### Q8: git push しても本番に反映されない

**チェック手順:**

1. **Render の Events タブ** で「Deploy started」が出ているか確認
2. 出ていない場合:
   - GitHub と Render の連携が切れている可能性
   - Render ダッシュボード → Settings → 「Reconnect repository」
3. Deploy started は出ているが Failed:
   - Logs タブでエラー内容確認
   - 通常は HTML/JS のシンタックスエラー

---

### Q9: 「Deploy failed」になった

**確認:** Render Logs タブで詳細を確認

**よくある原因:**
- HTML/JS の構文エラー（カンマ忘れ、クオート不一致）
- ファイルパスのタイプミス
- `render.yaml` の設定間違い

**緊急対処:** 前のコミットに戻す
```bash
git revert HEAD
git push
```

---

### Q10: 「Render の無料プランでスリープから戻るのが遅い」

**原因:** 15分間アクセスがないとスリープし、次回アクセス時に2〜3秒の遅延が発生

**対処オプション:**

**A. UptimeRobot で定期ping**（無料）
1. [UptimeRobot](https://uptimerobot.com) に無料登録
2. Render URL を5分間隔でping設定
3. → スリープしなくなる

**B. Starter プラン $7/月**
- Render ダッシュボード → サービス選択 → Settings
- Plan を Free → Starter に変更
- スリープなし、より高速

商用利用なら **Starter プランを推奨**。

## 🔵 makeshop 固有の問題

### Q11: makeshop のエディタが HTML を勝手に整形してしまう

**原因:** ビジュアルエディタモードで貼り付けている

**対処:**
1. **HTMLモード（ソースモード）** に切り替えてから貼り付け
2. それでも整形される場合 → **ブランクページB** を使う

---

### Q12: フリーページが表示できない（404）

**確認:**
- フリーページの「公開」設定がオンか
- URL のスペルが合っているか（半角英数字のみ）

---

## 📋 緊急時の連絡フロー

致命的な問題が発生した場合:

1. **応急処置:** Render ダッシュボードで Deploy History → 前のバージョンに「Rollback」
2. **詳細確認:** ブラウザの開発者ツールでエラー内容を確認・スクショ
3. **連絡:** 以下の情報を添えて開発元へ
   - 発生日時
   - 発生条件（どのページの何を押したか）
   - エラーメッセージ（スクショ）
   - ブラウザの種類とバージョン
   - スマホ/PC

## 🛡 予防策

### バックアップを取る習慣

大きな変更前には:
```bash
# ブランチで作業
git checkout -b backup-202612

# 変更前の状態を保存
git push -u origin backup-202612

# 何かあったら戻せる
git checkout backup-202612
```

### モニタリング設定

Render の無料機能:
- Deploy 失敗時のメール通知 → Settings → Notifications
- ヘルスチェック → 自動
