# GitHub 経由の更新方法

商品情報・価格・新製品追加などの更新方法です。
GitHubに push するだけで Render が自動で再デプロイします。

---

## 🔄 更新フローの全体像

```
[ローカルで編集]
       ↓
[git add → commit]
       ↓
[git push]
       ↓
[Render が自動検知]
       ↓
[1〜2分後、本番反映]
```

makeshop 側は **何もしなくてOK**。同じiframe URLで配信されます。

---

## 💡 更新方法は2通り

### A. ローカル編集 → push（推奨）

エディタで編集してから git push。
**履歴管理しやすい、まとめて編集できる**。

### B. GitHub のWebUIで直接編集

ブラウザだけで完結。
**価格1箇所変更など軽微な編集向け**。

---

## 🟢 方法A: ローカル編集 → push

### 必要なもの

- [Git](https://git-scm.com/) （PCにインストール）
- テキストエディタ（メモ帳でもOK、推奨: [VS Code](https://code.visualstudio.com/)）

### 初回のみ: リポジトリをクローン

```bash
# GitHubアカウントのフォークから clone
git clone https://github.com/YOUR-USERNAME/plush-puppy-coat-clinic.git

# フォルダに入る
cd plush-puppy-coat-clinic
```

### 編集サイクル（毎回）

```bash
# 1. 最新の状態を取得
git pull

# 2. ファイルを編集
# 例: index.html の PRODUCTS 配列を編集

# 3. 変更内容の確認
git status
git diff

# 4. コミット
git add .
git commit -m "feat: 新商品「ベリベリーシャンプー」を追加"

# 5. GitHub にプッシュ
git push
```

### コミットメッセージの推奨形式

| プレフィックス | 用途 | 例 |
|---------------|------|-----|
| `feat:` | 新機能・新商品追加 | `feat: ベリベリーシャンプーを追加` |
| `fix:` | バグ修正 | `fix: チョーケストラルの価格を修正` |
| `docs:` | ドキュメント更新 | `docs: 更新方法マニュアル改訂` |
| `style:` | デザイン調整 | `style: ボタンの色を調整` |
| `refactor:` | コード整理 | `refactor: PRODUCTS配列を整理` |

## 🟡 方法B: GitHub Web UI で直接編集

### 手順

1. GitHub のリポジトリページを開く（例 `https://github.com/YOUR-USERNAME/plush-puppy-coat-clinic`）
2. 編集したいファイル（例 `index.html`）をクリック
3. 右上の **鉛筆アイコン**（Edit this file）をクリック
4. ブラウザ上で編集
5. ページ下部の **「Commit changes」** をクリック
6. コミットメッセージを入力（例: `fix: シャンプーの価格を更新`）
7. **「Commit changes」** ボタンで確定

→ そのまま自動でRender再デプロイが走ります

## 📝 よくある更新例

### 例1: 商品価格の変更

「ナチュラル オールパーパス シャンプー」を ¥5,060 → ¥5,280 に変更:

1. `index.html` を開く
2. `Ctrl+F` で検索 → `pp_all_purpose` を入力
3. ヒットした箇所の `price:5060` を `price:5280` に書き換え
4. 保存 → commit → push

### 例2: 新商品の追加

`index.html` の `const PRODUCTS = [` から `];` の間に、以下のテンプレートを追加:

```javascript
  { id:'pp_new_product', cat:'shampoo', name:'新商品名', en:'New Product',
    price:0000, size:'500ml',
    url: ITEM_BASE + '000000000XXX',
    image: IMG_BASE + '000000000XXX_HASHHASH.jpg',
    matches:['daily'], coats:['all'],
    desc:'商品の特徴を1〜2文で。' },
```

書き換える箇所:
- `pp_new_product` → 内部ID（他と被らない名前）
- `新商品名` → 日本語商品名
- `0000` → 価格
- `000000000XXX` → plushpuppyjapan.com の商品ID
- `000000000XXX_HASHHASH.jpg` → 画像ファイル名
- `matches`, `coats` → 適合条件
- `desc` → 説明文

詳細は **`update-templates/products-template.md`** を参照。

### 例3: 販売終了商品の削除

該当商品の `{ ... },` のブロックを**まるごと削除**して push。

```javascript
// この行ごと削除
  { id:'pp_old_product', cat:'shampoo', ... desc:'...' },
```

## 🚨 重要: バックアップと安全運用

### 必ず守ること

- [ ] 大きな変更前には GitHub で **ブランチ**を切る
- [ ] 変更後は必ず Render のプレビューで動作確認
- [ ] コミットメッセージは具体的に書く（何月何日に何を変えたか後で見返せるように）

### ブランチを使った安全な編集

```bash
# 新しいブランチで作業
git checkout -b update-prices-202612

# 編集 → コミット
git add .
git commit -m "feat: 12月の価格改定"

# ブランチをGitHubにpush
git push -u origin update-prices-202612

# GitHub上でPull Requestを作成
# Renderは自動でプレビューURLを発行（変更内容を確認できる）
# 問題なければmainブランチにマージ → 本番反映
```

## 🆘 トラブルシューティング

### Q: push したのに反映されない

**A**:
- Render の「Events」タブでビルドが走っているか確認
- 「Build failed」と出ていれば、エラーログを開発元に共有
- 通常は1〜2分で反映、5分以上経っても反映されない場合は要連絡

### Q: 編集後に画面が真っ白になった

**A**:
- JavaScript 構文エラーの可能性（カンマ忘れ、クオート不一致など）
- ブラウザの開発者ツール（F12）→ Console でエラーを確認
- 直前のコミットに戻す: `git revert HEAD` → `git push`

### Q: 過去のバージョンに戻したい

**A**:

```bash
# 過去のコミット履歴を確認
git log --oneline

# 戻したいコミットの直前にハードリセット
# (注意: 以降のコミットは破棄されます)
git reset --hard <コミットハッシュ>
git push --force
```

不安な場合は開発元（Resolution Engine Inc.）へご相談ください。
