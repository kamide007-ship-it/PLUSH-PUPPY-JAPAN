# 製品情報の更新方法（詳細手順）

このマニュアルは、Plush Puppy Japan様が商品マスタを自社で更新できるようにするための手順書です。

---

## 更新できる項目

| 更新内容 | 必要なテキスト編集 | 難易度 |
|---------|------------------|--------|
| 価格の変更 | 1箇所の数字を書き換え | ★☆☆ |
| 商品サイズ（容量）の変更 | 1箇所の文字列を書き換え | ★☆☆ |
| 商品名の変更 | 1箇所の文字列を書き換え | ★☆☆ |
| 商品の説明文の変更 | 1箇所の文字列を書き換え | ★☆☆ |
| **新商品の追加** | 新規ブロックを追加 | ★★☆ |
| 商品の削除 | 該当ブロックを削除 | ★★☆ |
| 画像URLの変更（旧→新製品リニューアル時） | 1箇所のURLを書き換え | ★★☆ |
| 推奨ロジック（マッチ条件）の調整 | matches配列・coats配列の編集 | ★★★ |

---

## 製品マスタの構造

`index.html` を開いて、以下のような部分を探してください（だいたい1950行目あたり）:

```javascript
const PRODUCTS = [
  // ========== SHAMPOOS ==========
  { id:'pp_all_purpose', cat:'shampoo', name:'ナチュラル オールパーパス シャンプー', en:'Natural All Purpose Shampoo',
    price:5060, size:'500ml',
    url: ITEM_BASE + '000000000018',
    image: IMG_BASE + '000000000018_3qB3RoH.jpg',
    matches:['all_good','daily','low_shine'], coats:['all'],
    desc:'全犬種に使える定番ベース。日常ケアの基準点として最適です。' },
  ...
];
```

1製品が `{ ... }` の1ブロックです。最後のカンマ `,` を忘れないでください。

### 各フィールドの意味

| フィールド | 意味 | 例 |
|----------|------|----|
| `id` | 内部ID（変更しないこと） | `'pp_all_purpose'` |
| `cat` | カテゴリ | `'shampoo'` / `'conditioner'` / `'finishing'` / `'brush'` |
| `name` | 日本語商品名 | `'ナチュラル オールパーパス シャンプー'` |
| `en` | 英語名 | `'Natural All Purpose Shampoo'` |
| `price` | 価格（数字のみ・カンマ不要） | `5060` |
| `size` | 容量 | `'500ml'` / `'225g'` / `''`（ブラシ等） |
| `url` | 個別商品ページURL | `ITEM_BASE + '000000000018'` |
| `image` | 商品画像URL | `IMG_BASE + '000000000018_3qB3RoH.jpg'` |
| `matches` | 適合する症状・目的 | `['all_good','daily','low_shine']` |
| `coats` | 適合するコートタイプ | `['all']` または `['silky','cotton_toy']` 等 |
| `desc` | 説明文 | `'全犬種に使える定番ベース...'` |

---

## ケース1: 価格の変更（最も簡単）

例: 「ナチュラル オールパーパス シャンプー」の価格を ¥5,060 → ¥5,280 に変更する場合

### 手順

1. テキストエディタ（メモ帳でもOK）で `index.html` を開く
2. Ctrl+F で検索 → `pp_all_purpose` を入力
3. ヒットした箇所で `price:5060` を `price:5280` に書き換え
4. 保存
5. ホスティング先にアップロード

**注意**: 価格はカンマ無しの数字（`5,060` ではなく `5060`）

---

## ケース2: 新商品の追加

例: 新発売の「サマー クールシャンプー（¥4,500 / 500ml）」を追加する場合

### 必要な情報を集める

1. **商品ID**: plushpuppyjapan.com の管理画面で確認、または商品ページのURLから取得
   - URLが `https://www.plushpuppyjapan.com/view/item/000000000200` なら、ID は `000000000200`

2. **商品画像URL**:
   - plushpuppyjapan.com の該当商品ページを開く
   - 商品画像を右クリック → 「画像URLをコピー」
   - 例: `https://makeshop-multi-images.akamaized.net/PlushPuppyJP/itemimages/000000000200_AbCdEfG.jpg`

3. **カテゴリ**: シャンプー / コンディショナー / フィニッシング / ブラシ のいずれか

4. **適合症状・目的**: 下表から該当するものを選ぶ

### matches に指定できる値

| 値 | 意味 |
|----|------|
| `'low_shine'` | 艶不足 |
| `'low_moisture'` | 乾燥 |
| `'oily'` | 油分過多 |
| `'breakage'` | 毛切れ |
| `'matting'` | 毛玉 |
| `'low_volume'` | ボリューム不足 |
| `'static'` | 静電気 |
| `'yellowing'` | 白毛のくすみ |
| `'sensitive_skin'` | 敏感肌 |
| `'shedding'` | 換毛期 |
| `'soft_coat'` | コートが柔らかすぎ |
| `'all_good'` | 良好（維持ケア） |
| `'daily'` | 日常ケア向け |
| `'show_prep'` | ショー準備向け |
| `'white'` | 白毛向け |

### coats に指定できる値

| 値 | 意味 |
|----|------|
| `'all'` | 全コート対応 |
| `'silky'` | シルキーコート |
| `'cotton'` | コットン（全サイズ） |
| `'cotton_toy'` | コットン・トイサイズのみ |
| `'cotton_mini'` | コットン・ミニチュアサイズのみ |
| `'cotton_med'` | コットン・ミディアムサイズのみ |
| `'cotton_std'` | コットン・スタンダードサイズのみ |
| `'double'` | ダブルコート |
| `'harsh'` | ハーシュ（全サイズ） |
| `'harsh_mini'` | ハーシュ・ミニのみ |
| `'harsh_std'` | ハーシュ・スタンダードのみ |
| `'harsh_giant'` | ハーシュ・ジャイアントのみ |
| `'wavy'` | ウェービー |
| `'smooth'` | スムース |
| `'dachshund_smooth'` | ダックス（スムース）専用 |
| `'dachshund_long'` | ダックス（ロング）専用 |
| `'dachshund_wire'` | ダックス（ワイヤー）専用 |
| `'chihuahua_smooth'` | チワワ（スムース）専用 |
| `'chihuahua_long'` | チワワ（ロング）専用 |
| `'black_coat'` | 黒毛向け |

### 追加するコード

`PRODUCTS` 配列の最後の `];` のすぐ手前に、以下を追加します:

```javascript
  { id:'pp_summer_cool', cat:'shampoo', name:'サマー クールシャンプー', en:'Summer Cool Shampoo',
    price:4500, size:'500ml',
    url: ITEM_BASE + '000000000200',
    image: IMG_BASE + '000000000200_AbCdEfG.jpg',
    matches:['oily','daily'], coats:['all'],
    desc:'夏季限定。メントール配合でひんやりとした使用感が魅力です。' },
```

**注意点**:
- `id` は他の製品と被らない名前にする（`pp_` で始めるのが慣例）
- 行の最後に **カンマ `,`** を忘れずに
- 文字列は **シングルクオート `'`** で囲む

### 動作確認

1. 保存後、ブラウザで `index.html` を開く
2. 該当する条件を入力して「診断する」を押す
3. 新商品が推奨に出れば成功

## ケース3: 商品の削除（販売終了時）

該当商品の `{ ... },` の行を**まるごと削除**し、保存。

例:
```javascript
  // この行ごと削除
  { id:'pp_summer_cool', cat:'shampoo', ... desc:'...' },
```

## ケース4: 推奨ロジックの微調整

「あるシャンプーがおすすめされすぎる」「逆に出てこない」といった場合は、`matches` や `coats` を調整します。

### おすすめされすぎる場合

`matches` を絞る:
```javascript
// 修正前
matches:['low_shine','low_moisture','daily','show_prep'],

// 修正後 (daily を削除)
matches:['low_shine','low_moisture','show_prep'],
```

### おすすめに出てこない場合

`coats` に `'all'` を追加する、もしくは該当コートタイプを追加:
```javascript
// 修正前
coats:['silky'],

// 修正後
coats:['silky','wavy','all'],
```

## バックアップの取り方（編集前に必須）

編集する前に、必ず `index.html` のバックアップを取ってください:

1. `index.html` を右クリック → コピー
2. 同じフォルダで右クリック → 貼り付け
3. ファイル名を `index.html.backup-2026-05-24.html` などに変更

何かおかしくなったら、バックアップから戻せます。

## よくある間違い

### 1. カンマの抜け

```javascript
{ id:'pp_a', ... desc:'...' }    // ← カンマがない！
{ id:'pp_b', ... desc:'...' },
```

→ 全アプリが動かなくなります。必ず `},` でブロックを終わらせる。

### 2. クオートの不一致

```javascript
desc:"説明文"  // ❌ ダブルクオート
desc:'説明文'  // ✅ シングルクオート
```

### 3. URLの形式

```javascript
// ❌ 間違い（フルURLを直書き）
url:'https://www.plushpuppyjapan.com/view/item/000000000200',

// ✅ 正しい（ITEM_BASE を使う）
url: ITEM_BASE + '000000000200',
```

ITEM_BASE は `const ITEM_BASE = '...'` で先頭で定義されており、ベースURLが自動で前置されます。
